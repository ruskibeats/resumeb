# Russell Batchelor - Archetype Base Resumes

**Purpose:** Pre-built, strategically positioned resume templates for Russell's primary market lanes. Use these as the starting point when tailoring for a specific job, rather than rewriting from scratch every time.

---

## What Are Archetypes?

Russell has a 30-year career spanning infrastructure delivery, technical operations, programme management, and commercial leadership. No single CV can optimise for all of these. Instead, we build **Archetypes** - fully-realised CVs that position for one specific market lane with the right headline, evidence, skills, and keywords.

**The workflow:** When a JD comes in → Select the matching archetype (A, B, or C) → Fork it in Reactive Resume → Run the AI tailor with the JD → Apply → Export.

---

## Hybrid Umbrella Nesting Architecture

All three archetypes use the **Hybrid Umbrella Nesting** structure to prevent ATS "job-hopper" penalties:

1. **Hiloka Limited** (Jan 2010 – Present) as parent consultancy vehicle — all contract roles nested as child entries with `(Concurrent)` tags for overlapping dates
2. **London Borough of Lambeth** (Mar 2009 – Apr 2015) as standalone entry — too long-tenured to nest
3. **Earlier Career Highlights** — pre-2009 roles compressed to one-line proofs

This structure gives the appearance of 15+ years of steady consultancy engagement while detailing the diversity of project work underneath.

## Quick-Tailor Workflow

When a new job comes in, follow this workflow:

### Option A: Reactive Resume UI (Manual)
```
Step 1: Classify JD into Lane (A/B/C) using the Decision Tree below
Step 2: Open the Builder URL for the matching archetype
Step 3: Use "Save As" to fork a copy with a new name
Step 4: Use the AI Tailor button with:
       - The forked archetype as the base resume
       - The JD pasted as the target
Step 5: Review the AI suggestions and apply
Step 6: Export as PDF / DOCX
```

### Option B: Reactive Resume API (Programmatic)
Using the API at `http://192.168.0.178:3000/api/openapi` with API key header.

```bash
# Fetch archetype data
curl -s http://192.168.0.178:3000/api/openapi/resumes/{ARCHETYPE_ID} \
  -H "x-api-key: $API_KEY" | jq '.data' > archetype-data.json

# Import a tailored version
curl -s -X POST http://192.168.0.178:3000/api/openapi/resumes/import \
  -H "x-api-key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"title": "Tailored CV - Role Name", "data": <modified-resume-data>}'
```

### Decision Tree
```
Is the role focused on AI strategy, digital innovation, or AI operations?
  YES → Archetype D (Head of AI Operations)
  NO  → Does the role ask for "operational transformation" or "AI-enabled change" with a product/change management title?
    YES → Archetype E (Operational Transformation Manager — AI)
    NO  → Is the role an executive/director-level operations role with P&L?
      YES → Archetype C (Tech Ops Director)
      NO  → Is it a senior PM / delivery manager level role?
        YES → Archetype B (Senior Infra PM)
        NO  → Archetype A (Infra Programme Director)
```
