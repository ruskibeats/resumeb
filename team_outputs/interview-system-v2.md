## OPERATOR NOTES (for team-lead)

### How to run A → B → Fusion

**Step 1 — Interview A (Evidence Capture)**
- Run with candidate using the JSON payload `interview-a-payload.json`
- Purpose: verify, enrich and tag every major engagement with confidence level
- Each question offers MCQ options drawn from CV/interview evidence — candidate selects closest truth and can rewrite
- After completion: update evidence register with confidence tags (Verified / Partially verified / Needs evidence)

**Step 2 — Interview B (Work-Style & Role-Psych Fit)**
- Run immediately after, or in a separate session
- Purpose: map behavioural style to demands of the 3 target role types
- Scenario-based MCQ — no personality buzzwords
- Adaptive follow-up probes challenge overclaiming and test disconfirming evidence

**Step 3 — Fusion**
- CV writer reads both outputs together
- Use A outputs for CV bullet evidence and confidence flags
- Use B outputs to sharpen the profile statement and select emphasis (leadership vs delivery vs commercial)
- Red flags from B should inform which roles NOT to target or caveat in profile

**Critical guardrails for team:**
- Do not publish any bullet tagged Low confidence / Needs evidence without candidate re-confirmation
- Do not use clinical language (ADHD, neurodivergent) in CV copy without candidate explicit approval
- Challenge any heroic claim that lacks context + decision + impact triangle

b) Ask for more data before the meeting — I won't commit to a plan I can't stand behind  
c) Draft a high-level recovery narrative for Friday while simultaneously working the root cause in parallel  
d) Escalate to the client immediately — they need to know before Friday that the picture is more complex than a single plan can capture

*Role-fit signal:* (a) = execution-led, direct → strong for OpDir/PD | (b) = analytical, deliberate → strong for PM | (c) = hybrid → versatile | (d) = transparency-first → stakeholder strength, risk if premature
